# PRD — AUTORIZAÇÃO DE EXAMES

## 1\. Introdução

Este projeto visa criar o sistema **AUTORIZAÇÃO DE EXAMES**, uma plataforma web para controle administrativo de autorizações de exames e consultas, direcionado para funcionários administrativos de instituições de saúde.

Funcionários administrativos enfrentam dificuldade em controlar autorizações de exames e consultas, acompanhar gastos, organizar faturamento e gerar relatórios consolidados.

Este MVP resolve isso através de um sistema centralizado que permite registrar autorizações, controlar valores, acompanhar acréscimos de gastos e gerar relatórios financeiros por empresa, paciente, procedimento e período.

3.  Cadastro de empresas
    
4.  Cadastro de procedimentos (exames e consultas)
    
5.  Registro e gestão de autorizações

8.  Geração de relatórios financeiros detalhados

# ✅ Por que a Tabela de Itens da Autorização é Necessária

Atualmente sua modelagem permite:

-   1 autorização → 1 procedimento
    

Mas no mundo real da saúde é comum:

-   1 autorização → múltiplos procedimentos  
    Ex:
    
    -   Hemograma
        
    -   Raio-X
        
    -   Consulta
        
    -   Ultrassom
        

Se você não tiver uma tabela de itens:

-   ❌ Não consegue autorizar múltiplos procedimentos na mesma guia
    
-   ❌ Não consegue controlar quantidade por procedimento
    
-   ❌ Não consegue controlar valor individual por item
    
-   ❌ Faturamento ficará impreciso
    
-   ❌ Relatórios por procedimento serão inconsistentes
    

* * *

# 🔥 Modelagem Correta (Arquitetura Profissional)

Você deve transformar o relacionamento:

```
autorizacoes → procedimentos (1:N)
```

Em:

```
autorizacoes → itens_autorizacao (1:N)procedimentos → itens_autorizacao (1:N)
```

* * *

# 7\. 🚨 REGRAS ANTIFRAUDE

CA1: Não permitir alteração retroativa de data.  
CA2: Não permitir redução de valor após aprovação.  
CA3: Exclusão de autorização aprovada apenas por Administrador com justificativa obrigatória.  
CA4: Toda tentativa de violação deve ser registrada em logs.  
CA5: Não permitir alteração de faturamento fechado.  
CA6: Não permitir edição de itens após aprovação.  
CA7: Não permitir reduzir limite abaixo do valor já executado.

* * *

# 8\. REGRAS DE NEGÓCIO

RN-01 Quantidade > 0  
RN-02 Valor unitário > 0  
RN-03 total = soma dos itens  
RN-04 Faturamento = confirmados + glosados  
RN-05 Somente ADM cadastra bairros/povoados  
RN-06 Zona urbana exige bairro  
RN-07 Zona rural exige povoado  
RN-08 CPF duplicado bloqueia cadastro  
RN-09 Nome + data nascimento duplicado bloqueia cadastro
